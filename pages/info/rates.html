Soccer Rates

Last Update: June, 2020
Navigation
Description Introduction Definitions Adjust settings Examples Sources

Description

This is a small informational text about rates written by Termiii, from back then when the XSL still existed. Full credits go to him and the Sources at the bottom.

Introduction:

What are the best rate settings???? I don't know how often I've answered this already but there are no 'best rate settings'. Everyone has different settings which give them the 'smoothest feeling' on a server. You just have to find them and I hope this topic will help to do so.

Definitions:

First of all some definitions:

"rate" - maximum limit of bytes per second you can send to the server "cl_cmdrate" - determines how many physical packets per second the client will try and send to the server "cl_updaterate" - determines how often the server will attempt to send a physical packet to the client "choke" - number of updates not sent in either direction becausse the communication link is used to capacity "loss" - number of updates which were sent but never reached their destination "cl_interp" - tells your computer how fast to interpret the player and object positions sent to you by the server

Adjust settings:

Adjust your settings:

rate: You do not really need to worry about this one.. just set it to 25000.

cl_updaterate: Obviously it is the best to get as many updates from the server as possible because this gives you a more accurate picture of where objects and other players are. As the servers allow a maximum of 66.67 (I think) you should use cl_updaterate 67 to get the maximum.

cl_cmdrate: Basically the same as updaterate: You want as many updates per second as possible so you want this value as high as possible. The more updates you send to the server the better information it has about what you are doing. As the servers allow a maximum of 66.67 (I think) you should use cl_cmdrate 67 to get the maximum.

But since 67 is the standard value it's advised to not change it at all or, at least not to change it to any value less than 67.

However, IF you check your values via net_graph and the values for Choke and Loss (the values on the right) are differeing from each other pretty much, f.ex 50 and 64 while changing every second, you should change cmdrate and updaterate to the minimum value. If the lowest choke/loss is 50 you should change both, cmdrate and updaterate to 50, to prevent those values from changing all the time.

Usually this is all you have to change. (Atleast for me this has always been enough. I never changed anything about my interp...)

However some people stil get models not appearing smoothly.
If you do so you should use net_graph 3 and keep an eye on choke and loss. If you experience choke or loss during a match try using a lower updaterate and a lower cmdrate.


If you do not experience choke or loss but still have 'lags' then you should adjust your cl_interp.
The proper interp depends on your updaterate.
Your cl_interp should be: cl_interp = cl_interp_ratio/cl_updaterate (cl_interp_ratio is usualy 1 which means cl_interp = 1/cl_updaterate).

Examples

An example setup could look like this:

rate 25000
cl_cmdrate 67
cl_updaterate 67
cl_interp_ratio 1
cl_interp 0.014979 (= 1/66.67)

Hope this will help atleast some of you. I'm also not an expert with this, so if any1 knows something else feel free to add (or if something is wrong ofc).

NOTE: I personally never changed my rates when I was still active. I kept using the default one because in my opinion it is just a question of what you are used to. Changing your settings every day will not help. However, there are pretty solid settings, although they still got a dependency on the system you're running.

rate 25000
cl_interp 0
cl_interp_ratio 0
cl_cmdrate 67 // default value!
cl_updaterate 67 // default value!

If you're still experiencing lags with those settings, changing

cl_interp_ratio to -1

might be helpful
