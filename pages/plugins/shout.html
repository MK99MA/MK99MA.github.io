Shout-Plugin

Last Update: May 05, 2020 Version 1.5
Navigation
Description Credits Features Commands Download Installation Changelog

Description

Shout-Plugin is my first 'own' Sourcemod plugin. It tries to recreate the shout feature which was part of the original Eventscripts Soccermod. The Shout-Plugin however can be used as a standalone to play sounds ingame, either from a menu or (if enabled) by using hotkeys / commands. In addition it features a few additional settings to fine-tune the sounds globally or individually. Since it was more of a past-time project and I'm not an expert regarding SourcePawn, again, expect improvable code.

Credits

I wrote most of this plugin by myself, still I'd like to credit the following people and ressources:
Original Idea - Marco Boogers (and probably alot more who created similar plugins) Suggestion to recreate it - Subi Ressources - Sourcemod forums & Sourcemod API wiki

Features

Modes:

Ambient: Plays the shout from the position you used the command at. The closer people are to the location the louder they can hear it.

Team only: Plays the shout like any normal radio message to everyone in your team, independent of the distance. Due to the differences in broadcasting, the sounds will appear louder than in Ambient mode.

Everyone: Plays the shout to just like any normal radio message, but to everyone in the server (the teams?*), independent of the distance. Due to the differences in broadcasting, the sounds will appear louder than in Ambient mode.

Radius(Team): Plays the shout at full (set) volume to every member of your team in a changeable radius.

Radius(Everyone): Plays the shout at full (set) volume to everyone in a changeable radius.

Ingame Manager:

Add Shouts: The plugin will scan your sound folder and display any found (and not yet added) sounds. In the menulist you can then add sounds that should be usable. Duplicate soundnames will overwrite each other if their shoutname is identical.

Edit Shouts: Every added shout will be added to this menulist. You can change certain settings about each sound individually: volume, pitch, name. The given name will also determine the command to use (if activated)

Remove Shouts: Every added shout will be added to this menulist. You can remove any shout you don't want to use anymore with it.

Bulk Add: For whatever reason you want to add 50 shouts at once? Easy. Just throw them into the sound/shout folder and hit the Rescan option ingame. Every new sound should be added as a shout automatically (using its filename).

Fine-Tuning:

Volume: You can either change the global volume that will affect every shout, or set it individually per shout. The individual setting will always overwrite the global setting and can only be removed in the config file (for now).

Pitch: You can either change the global pitch that will affect every shout, or set it individually per shout. The individual setting will always overwrite the global setting and can only be removed in the config file (for now).

Cooldown: Set a cooldown any user has to wait before he can shout again. I'd advise you to never use a cooldown of 0 for the Ambient mode, since it can lead to an enormous shoutspam.

Radius: Only of relevance for the two radius modes. Determines the area in which any shout can be heard while those modes are active.

Usage:

Commands: If activated any shout can be used from the chat / bind by using !>shoutname< or sm_>shoutname< in chat or console.

Menu: By using !shout or sm_shout in chat or console you can access a menulist with every accesable shout. Please note that this list is sorted by the date the shout was added.

Misc:

Configuration: Any setting can be changed ingame using the settingsmenu or by editing the config files.

Integration: If SoMoE-19 is running on your server a shout menu shortcut will be added to Soccermods main menu, as well as settings menu shortcut to Soccermods settings menu.

Advert: Do you like Adverts Spam? Me neither, so the plugin will broadcast itself only once to a player whenever he joins. Additionally it can be toggled on or off in the settings menu.

Defaults: The Plugin comes with the most basic quakesounds set you can imagine.. And will add them to its shoutlist automatically after its first start if you install them.

Commands

Main Command:

!shout / sm_shout - Opens the Shoutlist menu.
!shoutset / sm_shoutset - Opens the Shout settings menu.
!>shoutname< / sm_>shoutname< - IF ENABLED: Quickly play any sound by using its name (as seen in the shoutlist) like a normal sourcemod command. Enabling the commands does require a map restart before the commands will work though.

Installation

I. Download the prerequisites:

To run Soccermod on your server you require the following plugins and files:
Please make sure to choose the right versions (Linux / Windows) depending on the OS of your server!
1. Metamod:Source 1.10 or higher 2. Sourcemod 1.10 or higher OPTIONAL BUT RECOMMENDED 3. Updater.smx and its prerequisites: Steamworks, Official Thread

II. Download Shout-Plugin:

Next download Shout-Plugin itself. There are two possible options to choose from:
1. Full Package (v1.5)
Includes the plugin, source and the training ball model. 2. Shout.smx (v1.5)
Includes only the latest plugin file for manual updates.

III. Additional Downloads (Optional):

Any custom sounds you want to use... (no links here for now)

IV. Extract Files:

After downloading the above files you have to extract its contents. Right click on each zip file and select 'Extract Here'. After extracting the zip files you should have the following folders wherever you extracted the zip files:
addons sound

V. Copy or Upload the files:

Copy or upload the folders to your server's 'cstrike' folder, for example:
D:\Servers\Counter-Strike Source\cstrike (local server) /home/cstrike (hosted server)

VI. Restart your server (If you freshly installed Sourcemod only!):

Finally, you just have to restart your server. After the restart Soccermod should be fully installed and loaded automatically.
If the plugin isn't loaded, first make sure that you are using the correct versions of Sourcemod and Metamod and the order structure is ok.
